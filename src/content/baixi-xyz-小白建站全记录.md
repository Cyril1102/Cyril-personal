## 一个请求的旅程

从你输入域名到看到网页，发生了什么：

```
你输入域名 → DNS（域名→IP）→ 防火墙（80/443端口放行）→ Nginx（Web服务器）→ HTML/CSS/JS → HTTPS → 备案 → 你的浏览器
```

### 什么是服务器？

服务器就是一台 **24小时不关机** 的电脑，放在腾讯云的机房里。你在上面放网页文件，别人访问域名时这台电脑就把网页发给对方。没有它，网站就没地方放。

### 什么是域名？

域名就是你网站的「门牌号」。服务器有一个数字 IP（`124.220.6.3`），但没人记得住。

域名就是给这串数字起个好记的名字。你输入域名，互联网自动找到对应服务器。

### 为什么选 .xyz？

- .com 好听的早被抢光，.cn 又贵限制又多
- .xyz 是新一代顶级域名，便宜、好看、全球通用
- Google 母公司 Alphabet 官网就是 `abc.xyz`，不丢人

## 第三步：配置 DNS

> 这一步最容易卡住，我在这里花的时间也最多。

### DNS 是域名和 IP 之间的对应关系

买了域名，域名只是一个名字，它还不知道你的服务器在哪。

DNS 解析就是告诉全世界：**`www.baixi.xyz` → `124.220.6.3`**。

负责这个工作的叫 **DNSPod**（腾讯云的 DNS 解析服务，基础功能免费）。

### 这里要做两步，而且缺一不可

#### ① 添加 A 记录（在 DNSPod 控制台）

| 记录类型 | 主机记录 | 记录值 | 作用 |
|----------|----------|--------|------|
| A | www | 124.220.6.3 | www.baixi.xyz → 服务器 |
| A | @ | 124.220.6.3 | baixi.xyz → 服务器 |

**什么是 A 记录？**

A 记录是 DNS 最基础的一种，含义就是「这个域名 → 这个 IP」。

`www` 管理 `www.baixi.xyz`，`@` 管理不带 www 的。**两条都要加。**

#### ② 改 DNS 服务器（域名注册管理页面）

域名注册时默认用的**不是** DNSPod。你要手动去「域名注册 → 管理 → DNS 服务器」改成：

```
sarah.dnspod.net
bear.dnspod.net
```

> 这是我踩的第一个大坑。
>
> 我在 DNSPod 加好了 A 记录，浏览器输入域名还是打不开。查了半小时才发现，问题出在 **DNS 服务器没改**。
>
> 域名默认用的不是 DNSPod，你在 DNSPod 配的一切，人家根本没在听。
>
> 好比：你跟小区保安说"快递送302室"，可你的小区压根没有保安——说了白说。

### 改完要等多久？

全球 DNS 同步一般 **1-48 小时**。我的大概一个多小时就全球生效了。

## 第四步：开放防火墙端口

> 这里我也卡了一次。

腾讯云 CVM 默认非常安全——安全到外面的人根本进不来。

### 端口可以理解成服务器的大门

```
你的服务器
├── 80  HTTP   已开放
├── 443 HTTPS  已开放
├── 22  SSH    仅自己使用
└── 其他端口   默认关闭
```

在腾讯云「安全组」添加规则：

| 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|
| TCP | 80 | 0.0.0.0/0 | HTTP 网站大门 |
| TCP | 443 | 0.0.0.0/0 | HTTPS 加密网站大门 |

**不开放会怎样？** 别人访问永远打不开，连接超时。服务器里配得再好，门没开，外面的人也进不来。

## 第五步：安装 Web 服务器（Nginx）

### Nginx：你网站的管家

**Nginx**（读作 Engine X）是一个 Web 服务器软件。它的工作：有人访问网站时，找到网页文件，发给对方。

就像一个管家——门口站岗，来人就问"你要去哪？"，带他去对应的房间。

配置里要告诉 Nginx 三件事：域名是什么、文件放在哪里、首页叫什么。

**不配置会怎样？** 文件放上去了也没人知道。没有 Nginx 配置，服务器不认识你的网站，访问就 404。

## 第六步：写网页

### 网页怎么来的

网页文件放 `/var/www/html/`，首页必须是 `index.html`。

### 什么是 HTML 和 CSS？

- **HTML**：像搭积木一样把文字、图片、链接拼在一起，搭出骨架。
- **CSS**：给骨架穿衣服——颜色、字体、间距、动画，全是 CSS。

## 第七步：申请 SSL 证书（HTTPS）

### HTTP vs HTTPS

| | HTTP（明文） | HTTPS（加密） |
|------|------|------|
| 传输 | 你 → 中间任何人 → 服务器 | 你 → 加密传输 → 服务器 |
| 安全性 | 全程可见，密码和消息都可能暴露 | 中间人看到的会是加密内容 |
| 浏览器 | 会提示「不安全」 | 地址栏会显示锁标识 |

> HTTP 更像明信片，HTTPS 更像封好的信封。

### 免费证书，一条命令搞定

**Let's Encrypt** 提供免费的 SSL 证书。用 Certbot 一条命令就可以申请：

```bash
certbot --nginx -d www.baixi.xyz -d baixi.xyz
```

它会自动完成域名验证、签发证书、修改 Nginx 配置，并把 HTTP 跳转到 HTTPS。

**到期怎么办？** 自动续期，每天凌晨3点检查：

```bash
0 3 * * * certbot renew --quiet --renew-hook 'systemctl reload nginx'
```

### 这里我踩了两个坑

1. **Certbot 401错误**：把**腾讯云 CAM 密钥**当成了**DNSPod API Token**，两者是完全不同的东西！
2. **HTTP 全变404**：Certbot 自动配置把重定向覆盖了。申请完记得检查。

**不配 HTTPS 会怎样？** 浏览器会提示不安全，内容也可能被窃听，有些浏览器甚至会直接拦截。

## 第八步：备案

### 备案是什么？为什么要备案？

中国法律规定：大陆服务器上的网站必须在工信部登记。

备案就是你告诉国家：「这个网站是我——xxx，叫xxx，用于xxx」。审核通过后，在国内才能正常访问。

**为什么不备案打不开？** 不是你的技术问题。运营商按工信部要求，主动拦截所有发往未备案网站的请求。服务器一切正常，但请求到不了就被拦下了。

### 流程和耗时

| 步骤 | 做什么 | 耗时 |
|------|------|------|
| ① | 登录腾讯云备案系统 | 2分钟 |
| ② | 填主体信息（姓名/身份证/手机） | 5分钟 |
| ③ | 填网站信息（域名/名称/用途） | 5分钟 |
| ④ | 传身份证照片 | 2分钟 |
| ⑤ | 人脸核验（微信扫码） | 1分钟 |
| ⑥ | 腾讯云初审 | 1-2个工作日 |
| ⑦ | 管局审核 | 3-20个工作日 |
| ⑧ | 通过，获得备案号 | 完成 |

### 网站名称有讲究！

不能叫「中国xxx」「全国xxx」，也不能叫「XX论坛」「XX社区」（需要专项审批）。

个人网站建议用：「白云深处」「baixi学习笔记」这类低调名字。

> 我的建议是先提备案。它最花时间，越早开始越省心。

## 踩坑汇总

| # | 问题 | 表现 | 原因 | 怎么办 |
|---|------|------|------|--------|
| 1 | 域名配了打不开 | 「找不到网页」 | DNS 服务器没改成 DNSPod | 买完域名第一时间去改 |
| 2 | 端口不通 | 连接超时 | 安全组没开 80/443 | 买完服务器先配安全组 |
| 3 | Certbot 报 401 | 证书申请失败 | 误用腾讯云密钥，不是 DNSPod Token | 去 DNSPod 单独创建 Token |
| 4 | HTTP 全变 404 | 页面全打不开 | Certbot 覆盖了重定向规则 | 申请后重查 Nginx 配置 |
| 5 | 国内打不开 | 「未备案」拦截 | 没备案 | 服务器买好就提交备案 |
| 6 | NS 记录不生效 | dig 无返回 | 改了 DNS 服务器需要等 1-48 小时 | 耐心等，用 dig 定检 |
