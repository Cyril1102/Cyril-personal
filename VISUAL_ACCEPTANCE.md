# 视觉验收

每次准备发布前，视觉验收必须同时通过三个关卡。构建成功只能说明代码可运行，不能替代视觉确认。

## 执行方式

```bash
npm run build
npm run visual:acceptance
```

验收产物位于 `artifacts/visual-acceptance/`，包括手机、桌面和 4K 三种尺寸的整页截图、关键组件截图、字号数据与验收报告。

## 关卡一：布局正确性

自动检查：

- 页面是否出现横向溢出。
- Hero、案例轮播、案例卡片、工作方法和教育经历是否超出视口。
- 关键组件是否缺失。
- 页面运行时是否出现错误。

## 关卡二：视觉比例

自动记录关键标题、正文和结果数字的实际字号，并检查案例轮播的字号上限与层级比例。4K 必须使用单独的密度预算，不能直接沿用普通桌面的判断结果。

自动通过后仍需人工查看：

- 标题是否抢占过多注意力。
- 结果数字与卡片尺寸是否协调。
- 留白、信息密度和阅读节奏是否符合当前设计方向。

## 关卡三：整页与局部截图

每种尺寸都要同时查看：

- `full-page.jpg`：判断整页构图、留白和章节节奏。
- `heroCarousel.png`：判断轮播字号与卡片比例。
- `hero.png`、`firstCase.png`、`method.png`、`education.png`：判断关键组件的局部层级。

最终验收记录必须分别写明：

- 布局正确性：通过 / 不通过。
- 视觉比例：通过 / 不通过。
- 人工截图复核：通过 / 不通过。
